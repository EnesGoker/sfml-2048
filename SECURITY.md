# Security Policy

## Supported Versions

Security fixes are provided for the latest `main` branch and the most recent tagged release.

## Reporting A Vulnerability

Please do not open public issues for suspected vulnerabilities.

Report privately with:

- affected version/commit
- platform (macOS/Linux/Windows)
- reproduction steps
- impact assessment
- optional proof-of-concept

Until a dedicated security mailbox is added, open a private channel with the repository owner and reference this file.

## Response Targets

- Initial acknowledgment: within 7 days
- Triage status update: within 14 days
- Fix timeline: depends on severity and reproducibility

## Disclosure

- Coordinated disclosure is preferred.
- A patch and release notes should be prepared before public disclosure.
